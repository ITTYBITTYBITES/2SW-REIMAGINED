# Witness Moment Interaction Design

Future Witness Moments should use hardware-neutral intents:

- Focus
- Enter
- Return
- Explore
- Reveal
- Reflect

The player should experience attention and discovery rather than controls. Touch, mouse, keyboard, controller, captions, and explicit access can map to the same moment intent without exposing a challenge-family UI.

Observation should not accept accidental answers. Reconstruction should use the appropriate accessible adapter. Investigation should offer optional assistance. Return should always pass through the Iris.
