# Legacy Experience Visibility Audit

## Purpose

Identify old Two Second Witness product assumptions still visible or reachable after the future ecosystem placeholders were added. This is classification only; nothing is deleted in this phase.

## Legacy elements currently visible or reachable

### Production Challenge Library / Experiences room

- Uses legacy terminology such as Challenge Library, Challenge Type, templates, and family cards.
- Reachable through the left Archive destination and the production destination host.
- **Disposition:** KEEP as a secondary advanced room; remove from first-session/root visibility, not from the product.

### Production Home/Product Hub concepts

- HomeV2, Play Now, Continue, Featured, Programs, and tab-style routes remain in production source.
- The Iris is now the visible root, but these concepts remain available to production routes/screens.
- **Disposition:** MODIFY into Director inputs, secondary Library/Program rooms, and recovery/accessibility paths.

### Family-first names

- Scene Investigation, Flash Words, Spot the Difference, Object Recall, and Pattern Recall remain visible in production Library/profile/history contexts.
- **Disposition:** KEEP in advanced/explicit contexts; REPLACE as the first Story Mode language with perception goals and Witness Moment names.

### Standalone tutorials

- Each family has a tutorial scene and production tutorial route.
- **Disposition:** MODIFY: first family introduction becomes a Story Mode beat; replay/tutorial access remains in Library/Training.

### Generic Observation / Recall / Result screen language

- Production screens still use direct challenge wording and conventional route presentation.
- **Disposition:** MODIFY framing and transitions while preserving mechanics/contracts.

### XP / Level / Achievement terminology

- Production profile screens expose conventional progression terms.
- **Disposition:** MODIFY presentation to Witness Rank, Chapters, Mastery, Discoveries, and Milestones while keeping production values/storage.

### Top bar / bottom tab navigation

- Existing AppShell and MainNavigation remain in source and can be visible in production rooms.
- **Disposition:** KEEP for explicit rooms/accessibility/compatibility; REPLACE as the default Iris navigation language.

## Legacy elements not currently primary

- Challenge selection is no longer the Iris center action.
- The Iris home does not show a card grid or challenge-family menu.
- Discover uses a constellation/floating-point model.
- Story Mode/Daily/Weekly placeholders use destination language rather than family names.

## Old terms to phase out of default Story Mode

- “Select a Challenge Type”
- “Choose a game mode”
- “Play a mini-game”
- “Challenge Library” as the first destination
- “Level select”
- “Results” as a terminal page
- “Tutorial” as a first-run page sequence

## Replacement language

| Legacy | Story Mode direction |
|---|---|
| Challenge Type | Witness Moment / perception ability |
| Scene Investigation | The Forgotten Museum / Notice what changes |
| Pattern Recall | A Memory Fragment Investigation / Recognize what connects |
| Results | Evidence / What remained hidden |
| XP/Level | Witness Rank / Rank Chapter |
| Achievements | Milestones / Discoveries |
| Profile | Your Iris / Witness Record |
| Settings | Calibration / Tune the instrument |

## Removal/archival candidates

No files are removed in this audit. Eventual candidates are documented in `OBSOLETE_SYSTEM_INVENTORY.md`:

- legacy sample Witness fallback;
- prototype destination drawings after host parity;
- old root AppShell assumptions after compatibility testing;
- historical migration documents after archive policy;
- generated artifacts.

## Acceptance requirement

The old system can remain technically reachable, but it must not be the first thing a new user sees or the only way to understand the product.
