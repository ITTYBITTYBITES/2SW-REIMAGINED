# Challenge Rebuild Direction

## Purpose

Existing challenge families are tools inside Witness Moments, not primary navigation destinations. This document defines what survives and how future presentation rebuilds should frame those mechanics.

## Preserve across every rebuild

- family identity and purpose;
- seeded generation;
- fairness validator and fallback;
- difficulty/exposure policies;
- production interaction adapters;
- family-owned scoring and results;
- progress/mastery/history contracts;
- accessibility and timing accommodation;
- content versioning and replay metadata.

## Scene Investigation → The Forgotten Museum

**Old name:** Scene Investigation

**Future experience framing:** The Forgotten Museum

**Internal mechanic:** observe a rich scene, recall one detail, reveal what was missed.

Preserve scene generation, question types, target evidence, and scoring. Rebuild the framing, authored atmosphere, observation pacing, and evidence reveal.

## Pattern Recall → A Memory Fragment Investigation

**Old name:** Pattern Recall

**Future experience framing:** A Memory Fragment Investigation

**Internal mechanic:** watch a connected path or symbol sequence, reproduce order.

Preserve path/sequence generation, timing, sequence adapter, and prefix evidence. Rebuild the reason the pattern matters, visual language, and reward/Archive meaning.

## Object Recall → The Empty Table

**Old name:** Object Recall

**Future experience framing:** The Empty Table

**Internal mechanic:** remember what was present, missing, grouped, or at the ends.

Preserve the reviewed object pool, set/missing/row/bookend templates, multi-selection adapter, and evidence data. Rebuild the spatial composition and the emotional transition from a filled field to absence.

## Spot the Difference → The Room Between Moments

**Old name:** Spot the Difference

**Future experience framing:** The Room Between Moments

**Internal mechanic:** compare two states and identify one semantic change.

Preserve paired/sequential mutation validation, spatial targets, accessible single-choice fallback, and result evidence. Rebuild the change as an intentional witness clue rather than a puzzle-grid task.

## Flash Words → The Vanishing Signal

**Old name:** Flash Words

**Future experience framing:** The Vanishing Signal

**Internal mechanic:** recognize a word/sequence/position before it disappears.

Preserve reviewed word content, reading comfort, recent-word prevention, timing policies, and exact comparison. Rebuild the framing so speed supports attention without becoming the entire point.

## Rebuild layers

### Layer 1 — Story framing

- chapter/rank context;
- reason this moment matters;
- Iris voice/visual invitation;
- no family-first menu.

### Layer 2 — Observation presentation

- cinematic entry;
- family renderer with production-quality visual language;
- shared attention-window framing;
- accessibility equivalent.

### Layer 3 — Recall presentation

- family adapter remains correct;
- copy feels like a witness question;
- result input is forgiving and legible.

### Layer 4 — Evidence/reward

- reveal what was there/changed/connected;
- connect evidence to Archive;
- update rank/mastery/progression;
- return with Iris memory.

## What should be replaced

- challenge-family-first labels as the default player journey;
- generic shared result copy;
- standalone tutorial-first framing;
- unrelated page-to-page transitions;
- presentation that hides the evidence meaning;
- final artwork that does not support the exact target/recall contract.

## What should not be replaced

- generator/validator contracts;
- family-owned scoring;
- production saves/progress;
- accessibility adapters;
- content manifests;
- seeded/replay behavior;
- ChallengeSessionService lifecycle.

## Acceptance principle

A rebuilt challenge is successful when a player can describe the moment’s perception goal without needing to know its internal family name, while the existing production runtime continues to generate, validate, score, save, and explain the result correctly.
